# 墨刀AI 平台开发规范与防错指南

## 1. 运行时稳定性 (防白屏/编译失效)
- **依赖项校验**：引入新的 UI 组件库（如 Radix UI）时，务必先检查 `package.json` 是否包含该依赖。如果缺失，必须立即使用 `npm install` 安装，防止出现 `Failed to resolve import` 编译错误。
- **图标引用完整性**：从 `lucide-react` 导入图标时，需确保顶部导入列表与组件内使用的名称完全一致。
- **JSX 结构保护**：在进行大幅度代码块替换（尤其是涉及多个嵌套 `div` 的区域）后，必须扫描上下文以确保所有标签正确闭合。

## 2. 界面视觉一致性
- **地址栏规范**：预览区的地址框应保持收敛感（建议 `max-w-[200px]` 左右），并具备 Home 键与竖线物理分隔。
- **间距控制**：顶部面板的“发布”按钮与操作图标组间距应保持紧凑（如 `ml-1`），避免视觉断层。

## 3. 开发环境配置
- **端口固定**：项目开发端口应固定在 `5174`（在 `vite.config.ts` 中配置），避免因端口自动增量导致预览失效。
